Change Criteria
High
  • what the service provide to an external customer is significantly modified i.e a change to concept
  • change affects services at multipale sites
  • novel funcuality is to be provided
  • new instrument flight producer concept introduction
Meduim
  • change does not implement new funcuality, but it changes the operator's Humen Machine Interface (HMI)
  • change affects a single service at a single sites
  • change is a high significate but has been implemented before at another site
  • change has the potintial to undermine the accurecy of the data provided
  • new cyber security control with significate change which require system re-design is implemented
  • change affect more than two work-related fatigue factors
Low
  • It is an update to operational and technical procedures /document
  • it is a hardware refresh
  • it is installation of operating system(Batteries, USP, etc) without direct affect on the main system
  • it is a software update without functional change
  • it is an update to exist cyber-security control or a new cyber-security control is implemented with no significate change
  • it is a routine change that have been performed many times or are planned to be performedmany times and procedures are in place to manage this type of change
